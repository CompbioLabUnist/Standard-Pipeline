# Data pre-processing for variant discovery

This module follows the GATK data pre-processing workflow:
https://gatk.broadinstitute.org/hc/en-us/articles/360035535912-Data-pre-processing-for-variant-discovery

For each paired-end DNA sample, the pipeline maps reads, sorts BAM files, marks
duplicates, calculates BQSR, and applies BQSR. The final BAM is suitable for
downstream somatic and germline variant calling.

## Workflow objectives and result interpretation

The objective of this module is to convert raw paired-end DNA FASTQ files into
a coordinate-sorted, duplicate-marked, base-quality-recalibrated BAM file for
variant discovery. The final output from this module is:

```text
<sample>.Sort.MarkDuplicates.BQSR.bam
```

Use this final BAM as input for downstream somatic variant calling, germline
variant calling, and copy-number analysis.

### 1. Alignment

Alignment maps FASTQ reads to the reference genome listed in `[REFERENCES]
fasta`. This module provides two aligner entry points:

- `02_1_BWA.py`: uses `bwa mem` and is the recommended path for this pipeline.
- `02_2_Bowtie2.py`: uses Bowtie2 as an alternative aligner.

The aligner receives two paired FASTQ files and writes an unsorted BAM:

```text
<sample>.bam
```

Main purpose:

- determine where each read pair likely originated in the reference genome
- retain read-pair information needed by variant callers
- attach read-group metadata including sample name, library, platform, and
  sequencing center

How to interpret the result:

- This BAM is an intermediate file, not the preferred downstream input.
- It contains mapped and possibly unmapped reads.
- It is not guaranteed to be coordinate-sorted.
- Poor alignment rates can indicate low read quality, contamination, wrong
  reference genome, wrong sample input, or library problems.

### 2. Sort

Sorting orders alignments by genomic coordinate using `samtools sort`.

Input:

```text
<sample>.bam
```

Output:

```text
<sample>.Sort.bam
```

Main purpose:

- place reads in chromosome and position order
- make the BAM compatible with GATK tools and most genome browsers
- allow efficient random access when an index is available

How to interpret the result:

- This is still an intermediate file.
- Reads from the same genomic region are now grouped together.
- The script runs `samtools sort` with `--write-index`, so an index file is
  also generated by samtools.

### 3. Mark duplicates

Duplicate marking identifies reads that likely came from the same original DNA
fragment. This pipeline uses GATK `MarkDuplicatesSpark`.

Input:

```text
<sample>.Sort.bam
```

Outputs:

```text
<sample>.Sort.MarkDuplicates.bam
<sample>.Sort.MarkDuplicates.metrics
```

Main purpose:

- detect PCR duplicates and optical duplicates
- mark duplicate reads in the BAM instead of deleting them
- produce duplication metrics for library-complexity review
- prevent duplicate reads from being counted as independent evidence during
  variant calling

How to interpret the result:

- Duplicate reads are flagged in the BAM. They remain in the file unless a
  downstream tool chooses to ignore them.
- The metrics file summarizes duplicate counts and duplicate fractions.
- High duplication can be expected in targeted or exome sequencing, but very
  high duplication may indicate low input DNA, over-amplification, or low
  library complexity.

### 4. Base quality score recalibration

Base quality score recalibration (BQSR) models systematic sequencing-error
patterns in base quality scores. This pipeline uses GATK `BaseRecalibrator`
with known variant resources configured in `[REFERENCES] sites`.

Input:

```text
<sample>.Sort.MarkDuplicates.bam
```

Output:

```text
<sample>.Sort.MarkDuplicates.BQSR.table
```

Main purpose:

- compare observed mismatches against known variant sites
- estimate systematic quality-score bias by cycle, context, and reported
  quality
- create a recalibration table used by the next step

How to interpret the result:

- The BQSR table is not a BAM file and does not contain reads.
- It stores the recalibration model calculated from the duplicate-marked BAM.
- Incorrect or missing known-sites resources can produce poor recalibration, so
  the known-sites files must match the reference genome build.

### 5. Apply BQSR

`ApplyBQSR` applies the recalibration table to the duplicate-marked BAM and
writes the final analysis-ready BAM.

Inputs:

```text
<sample>.Sort.MarkDuplicates.bam
<sample>.Sort.MarkDuplicates.BQSR.table
```

Output:

```text
<sample>.Sort.MarkDuplicates.BQSR.bam
```

Main purpose:

- adjust base quality scores using the BQSR model
- preserve alignment positions and duplicate flags
- produce the final BAM used by downstream variant discovery modules

How to interpret the result:

- This is the main output of `02_Data_pre-processing_for_variant_discovery`.
- It should be used as input for `03_Somatic_short_variant_discovery`,
  `04_Germline_short_variant_discovery`, and
  `09_Somatic_copy_number_variant_discovery`.
- The file should have the same genomic alignments as the duplicate-marked BAM,
  but with recalibrated base qualities.

## Setup

Run commands from this directory:

```bash
cd 02_Data_pre-processing_for_variant_discovery
ln -snfv ../pipeline_utils.py .
```

Input FASTQ names are expected to contain `_DNA`; the sample name is derived
from the part before `_DNA`.

## Execute with the example wrappers

Edit the FASTQ paths in the wrapper before running.

BWA:

```bash
bash 02_1_BWA.bash
```

Bowtie2:

```bash
bash 02_2_Bowtie2.bash
```

## Execute BWA directly

Generate the SLURM scripts without submitting:

```bash
python3 -B 02_1_BWA.py --dryrun \
    "$(realpath ../00_Data/WES/merge/C001_TN_DNA_R1.fastq.gz)" \
    "$(realpath ../00_Data/WES/merge/C001_TN_DNA_R2.fastq.gz)" \
    "$(realpath .)"
```

Submit the jobs:

```bash
python3 -B 02_1_BWA.py \
    "$(realpath ../00_Data/WES/merge/C001_TN_DNA_R1.fastq.gz)" \
    "$(realpath ../00_Data/WES/merge/C001_TN_DNA_R2.fastq.gz)" \
    "$(realpath .)"
```

## Execute Bowtie2 directly

```bash
python3 -B 02_2_Bowtie2.py \
    "$(realpath ../00_Data/WES/merge/C001_TN_DNA_R1.fastq.gz)" \
    "$(realpath ../00_Data/WES/merge/C001_TN_DNA_R2.fastq.gz)" \
    "$(realpath .)"
```

Arguments:

- `input`: two paired FASTQ files.
- `output`: output directory for BAM files, generated job scripts, and logs.
- `--config`: config INI file. Defaults to `../config.ini`.
- `--dryrun`: create scripts in `sh/` without calling `sbatch`.

## Submitted job order

BWA mode:

1. `1-1.BWA`
2. `1-2.Sort`
3. `1-3.MarkDup`
4. `1-4.BQSR`
5. `1-5.ApplyBQSR`

Bowtie2 mode:

1. `2-1.Bowtie2`
2. `2-2.Sort`
3. `2-3.MarkDup`
4. `2-4.BQSR`
5. `2-5.ApplyBQSR`

Each job is submitted with an `afterok` dependency on the previous job.

## Expected outputs

For sample `C001_TN`, outputs include:

- `C001_TN.bam`: unsorted alignment BAM from BWA or Bowtie2.
- `C001_TN.Sort.bam`: coordinate-sorted BAM.
- `C001_TN.Sort.MarkDuplicates.bam`: sorted BAM with duplicate reads marked.
- `C001_TN.Sort.MarkDuplicates.metrics`: duplication metrics from
  `MarkDuplicatesSpark`.
- `C001_TN.Sort.MarkDuplicates.BQSR.table`: recalibration table from
  `BaseRecalibrator`.
- `C001_TN.Sort.MarkDuplicates.BQSR.bam`: final recalibrated BAM for
  downstream analysis.
- generated scripts in `sh/`
- SLURM logs in `stdeo/`
